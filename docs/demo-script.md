# 录屏脚本

## 1. 需求理解

说明本题重点不是 UI，而是 AI 工具使用、MVP 拆解、核心流程可运行和边界清楚。

## 2. MVP 定义

第一版做内部员工助手：

- 普通聊天。
- 多轮上下文。
- 自动工具调用。
- mock 模式。
- 本地可启动。

不做登录、数据库、RAG、流式输出和生产级权限。

## 3. AI 协作过程

展示如何让 AI 帮助：

- 拆解需求。
- 设计目录。
- 生成实现提示词。
- 审查并收敛范围。
- 修复构建或测试问题。

## 4. 本地启动

```bash
pnpm install
pnpm dev
```

打开 http://localhost:5173。

## 5. 演示用例

普通对话：

```text
你好，你能做什么？
```

HR 工具调用：

```text
公司年假政策是什么？
```

多轮追问：

```text
那远程办公时也适用吗？
```

待办工具：

```text
帮我创建一个待办：明天提交报销
```

时间工具：

```text
现在几点？
```

## 6. Mock 模式

说明没有 `.env` 和 API Key 时仍默认 mock 模式运行。

## 7. 限制和后续优化

说明当前限制：

- 内存会话重启丢失。
- 工具路由是演示级规则。
- 真实模型模式未做完整 function calling。

后续优化：

- 数据库持久化。
- 真实 function calling。
- 权限和审计。
- 流式输出。
